Part One: Programming Fundamentals

Language Basics

Programming is the activity of creating an ordered set of instructions that a computer can execute to perform specific tasks. Unlike natural language, which can be ambiguous, communicating with a computer requires the use of a precise formal language.

Through the writing of source code (or simply "code"), the programmer implements algorithms, which are logical procedures that transform input data into the desired output results.

The Integrated Development Environment

An "Integrated Development Environment" (IDE) consists of all the necessary tools to write, execute, debug a program, and display the results.

The Matita user interface presents three main parts:

  • Editor: The area dedicated to writing the source code.
  • Graphics Area (Canvas): The visual space where drawings and animations are rendered.
  • Console: The textual interface for message output and diagnostics.

At the top of the editor, there is a menu with some functional buttons:

  • Open / Save: Allow opening and saving Matita programs as text files with a .txt extension.
  • Download Matita: Allows saving the entire development environment as a single HTML file on your computer. This lets you work locally, without the need for an internet connection. Furthermore, we will see that using a local copy makes it possible to work with images saved on your device.
  • Language Toggle (EN/IT): Changes the interface language, but that's not all. It automatically translates the entire source code written in the editor from English to Italian (or vice versa), preserving comments and text strings intact.
  • Help: Opens the technical documentation in a new window.
  • Run: The main button. It sends the command to the computer to interpret and execute the instructions.

Keyboard Shortcuts

To speed up writing and testing code, the IDE supports several useful key combinations:

  • Ctrl + Enter: Equivalent to the Run button. Immediately starts interpreting the current program.
  • Ctrl + Space: Activates the automatic suggestions (IntelliSense) system. If typed while writing a word, it opens a list of commands compatible with what has been typed.
  • TAB: If the suggestions menu is open, the TAB key accepts and automatically inserts the selected command into the editor. Furthermore, TAB is useful to indent the code and keep it organized.
  • ESC: Closes the suggestions window or other modal windows (like the Graphics Area if in the foreground), returning control to the editor.

The First Commands: write and clear

The write command sends text messages to the console. Consider the following example:

write "Hello world"

In the console, the output will appear as:

Hello World, preceded by the system prompt :).

Matita executes the program by interpreting the instructions sequentially, from the first to the last:

write "A"
write "B"

The previous code will produce the output AB.

To insert a line break, the newline command is used:

write "A"
newline
write "B"

The special escape character \n performs the same function as newline within a string. The previous program is equivalent to:

write "A\n"
write "B"

It is possible to concatenate multiple strings using the + operator. Example:

write "Hello " + "World.\n"

To clear the console of all previous text, the clear screen command is used.

clear screen
write "First line."
newline
write "Second line."

A fundamental syntactic element is the # character, which indicates the start of a comment. The text following this character is ignored by the interpreter. Comments are useful for documenting the code:

# this is a comment.
put 31 in DaysInMarch # number of days in the month of March
# Let's show how many days March has:
write "March has " + DaysInMarch + " days."
Syntax highlighting and suggestions
The Matita IDE uses a color code to highlight the different syntactic elements of a program. Additionally, it offers suggestions as a mnemonic aid for using constructs.

Multiple commands on one line

To insert multiple instructions on the same line, they must be separated with a semicolon (;):

write "hello."; newline

Single command across multiple lines

Should a command be too long for a single line, it is possible to break it using a backslash \ at the end of the line. The interpreter will consider the lines separated by \ as a single instruction:

write \
"instruction on two lines"

Variables

A variable represents a memory area identified by a symbolic name (label), intended to hold a value (number, text, or complex data structures). Using variables allows writing programs that operate on dynamic data, abstracting from the specific value held inside.

To declare a variable and assign it a value (initialization), the command is used: put ... in VariableName.

Consider the following example:

put 5 in A
put "Computer Science" in B
write "The value in A is " + A
newline
write B

The produced output will be:

:) The value in A is 5
:) Computer Science

Note that text strings must be enclosed in quotation marks, for example "Computer Science".

To read the content of a variable, it is sufficient to write the variable name within an expression (as in the previous example with A and B). In other words, the name is the identifier that allows accessing the stored value.

In Matita, variable names are case-insensitive: Name, name, nAmE, NAME all refer to the same memory location. A valid identifier can contain numbers but must start with a letter. Accented or special characters are not allowed to define the name.

To update the value of a variable, the put command is used again.

put 100 in health
write "Initial health: " + health
newline
put health - 15 in health # The player takes 15 damage
write "Status updated. The health is now: " + health

User input: get

The get ... saying ... command triggers a dialog window (prompt) that allows the user to enter data via the keyboard. The entered value is stored in the specified variable.

get PlayerName saying "Enter username:"
write "Registered name: " + PlayerName + ". Procedure started."

Mathematics

Arithmetic Operations and Precision

Expressions can be evaluated and directly assigned to variables.

put 10 in A
put 5 in B
put A+B in Sum
put A*B in Product
write "A + B = " + Sum; newline
write "A * B = " + Product

The mod operator ("modulo") returns the remainder of an integer division. For example, 10 mod 3 returns 1.

clear screen
put 39 in A
put 7 in B
put A mod B in C
write "The remainder of " + A + "/" + B + " is " + C

Operators and Expressions

In Matita, an expression is a combination of operands (numbers, texts, lists, points) and operators. The result can be assigned to a variable or used as a logical condition.

  • Boolean values: true, false
  • Logical operators: and, or, not
  • Comparisons:
    • Equality: equals, is
    • Inequality: different (or !=)
    • Numeric comparisons: <, >, <=, >=
  • Arithmetic operators: +, -, *, /
  • Remainder: mod
  • Power: ^ (e.g. 2^3)
  • Negation (unary): -X

Power example:

put 2^3 in Cube
write Cube # Output: 8

Constants and Mathematical Functions

Matita includes various ready-to-use constants and mathematical functions. A complete list of predefined functions is available in Appendix A.

  • Constants: pi (or π), infinity, nan (Not a Number)
  • Trigonometric and calculus functions: abs(x), sin(x), cos(x), squareroot(x), etc.
  • Rounding functions:
    • round(x): Rounds to the nearest integer.
    • trunc(x): Removes the decimal part without rounding (truncation towards zero).
    • ceil(x): Rounds to the next higher integer (upwards).

Rounding example:

write trunc(3.9) # Output: 3
newline
write ceil(3.1)  # Output: 4

Points and Coordinates: point

Besides simple numbers, Matita allows working with Cartesian coordinates via the point object. This is essential for graphics, but it can also be used for vector or geometric calculations.

  • point(x, y): Creates a point object with the specified coordinates.
  • x of P / y of P: Accesses the horizontal and vertical coordinate of the point P, respectively.
put point(10, 50) in P1
put point(20, 50) in P2

# Distance calculation (simplified, X-axis only)
put (x of P2) - (x of P1) in Distance
write "Horizontal distance: " + Distance 
# Output: :) Horizontal distance: 10

Generating Random Numbers

Generating random numbers is essential for simulations and games. Matita offers several options via the random and dice commands:

  • random(): Without arguments, it returns a real number between 0.0 (inclusive) and 1.0 (exclusive).
  • random(MAX): Returns a real number between 0.0 and MAX.
  • random(MIN, MAX): Returns a real number between MIN and MAX.
  • dice(N): The command simulates rolling a die with N faces, so it returns an integer number between 1 and N, inclusive.

Examples:

put random() in Probability # e.g. 0.453
put random(100) in Percentage # e.g. 87.21
put random(10, 20) in Temperature # e.g. 14.5
put dice(6) in Toss # e.g. 4 (gives an integer between 1 and 6)

Defining Numeric Precision

To control the number of decimal places displayed in the write command, the digits command can be used. This command only affects the display of the output in the console, not the internal value of the variable.

digits 2
put 10 / 3 in Result
write Result # Output: 3.33
newline
digits 5
write Result # Output: 3.33333

Scientific Notation: scientific

When Matita handles very large numbers, the system can automatically switch to displaying in scientific notation to improve readability. The format used is "number e exponent".

By default, the threshold for enabling scientific notation is 1e6.

This threshold can be modified using the scientific(N) command.

If a value for digits is also set, this will affect the number of digits shown in the mantissa.

# Let's set a low threshold to see the effect immediately
scientific(100) 
put 150 in Value

write Value 
# Output: :) 1.5e+2

# Let's change the decimals displayed in the mantissa
digits 4
write Value
# Output: :) 1.5000e+2

Operator Precedence

  1. or
  2. and
  3. Comparisons (equals, different, <, etc.)
  4. +, -
  5. *, /, mod
  6. ^ (Power)
  7. Property or element access (e.g., x of P, item N of LIST)

Lists

The list is the fundamental data structure in Matita that allows storing an ordered sequence of elements. Unlike a simple variable, which holds a single piece of data, a list holds multiple data accessible via their position, called index.

Lists in Matita are heterogeneous, meaning they can contain different data types, and dynamic, meaning their size can vary during execution.

Creation

To instantiate a list, group the elements in parentheses:

put (10, 20, 30) in myList

For greater semantic clarity, the list keyword can be used:

put list(10, 20, 30) in myList
put list() in EmptyList

Operations on Lists

Matita provides a set of instructions for manipulating lists.

Accessing Individual Elements

To read a value at a specific position, the item ... of ... syntax is used.

Indexing in Matita is 1-based: the first element is at index 1, the second at index 2, etc.

put (100, 200, 300) in data
put item 2 of data in Value # Value now contains 200
write Value # Prints 200

Searching for an element: To find the position of a specific value within the list, the indexof(list, value) command is used. If the element is found, its index is returned (starting from 1), otherwise 0 is returned.

put ("Apple", "Pear", "Banana") in Fruits
put indexof(Fruits, "Pear") in Position
write "The Pear is at position " + Position # Output: 2

Determining the Dimension

To get the cardinality of the list (number of elements), use the dimension of operator.

put ("red", "green", "blue", "yellow") in ColorList
put dimension of ColorList in N # N now contains 4
write ColorList + " are " + N + " colors\n"
write "the second color is " + item 2 of ColorList

The output will be:

:) red,green,blue,yellow are 4 colors
:) the second color is green

Maximum and minimum of a list of values

Matita offers functions that return the extreme values within a numeric list.

  • min(list): Returns the lowest value.
  • max(list): Returns the highest value.

When working with "hybrid" lists that contain both numbers and text, or when it is necessary to know not only the extreme value but also its position (index), advanced functions are used:

  • valmin(list) / valmax(list): Calculate the minimum and maximum ignoring non-numeric values (such as strings or nulls).
  • argmin(list) / argmax(list): Return the index (the position) where the minimum or maximum value is located, respectively.

Below is a complete example of how to use these functions to analyze a data set.

# Let's introduce the data to work on.
# Note that the heights list is hybrid, because it doesn't contain 
# only numeric values: the header, in fact, is a string.

put (("Name", "Height"),\ 
("Marco", 175), ("Anna", 182), ("Luca", 185), ("Giulia", 176), ("Paolo", 189),\
("Sofia", 168), ("Davide", 178), ("Elena", 165), ("Matteo", 181),\
("Chiara", 173) ) in Group

# We create the list of heights
put () in HeightsList # Initialization
for each P in Group you do ( add item 2 of P to HeightsList )

# Let's find where the shortest and tallest members of the group are respectively:
put argmin(HeightsList) in MinIndex
put argmax(HeightsList) in MaxIndex

# Let's retrieve the names:
put item 1 of (item MinIndex of Group) in ShortName
put item 1 of (item MaxIndex of Group) in TallName

# Finally, output:
clear screen
write "Shortest member: " + ShortName + " (" + valmin(HeightsList) + ")"; newline
write "Tallest member:  " + TallName  + " (" + valmax(HeightsList) + ")"

Expected output:

:) Shortest member: Elena (165)
:) Tallest member: Paolo (189)

Modification

Lists are mutable through specific commands.

Adding elements: The add command inserts a new element at the end of the list. Syntax: add <value> to <listName>

put list(1, 2) in numbers
add 3 to numbers # numbers is now (1, 2, 3)

Inserting at a specific position: To insert a value at an arbitrary position, shifting subsequent elements, use insert ... in item ... of .... Syntax: insert <value> in item <index> of <listName>

put ("A", "C") in Letters
insert "B" in item 2 of Letters
write Letters
# Output: :) (A, B, C)

Removing at the head or tail:

put ("red", "green", "blue", "yellow") in ColorList
remove first item from ColorList
write ColorList
newline
remove last item from ColorList
write ColorList

Output:

:) (green, blue, yellow)
:) (green, blue)

Removal by index:

clear screen
put ("A","B","C","D") in LetterList
remove item 3 from LetterList
write LetterList

Output: :) (A, B, D)

Updating an element: To overwrite a value at a given position, combine put with the element access.

put (10, 20, 30) in Data
write Data; newline
put 99 in item 2 of Data
write Data

Output:

:) (10, 20, 30)
:) (10, 99, 30)

Iteration

To execute a block of instructions for each element of the list, the for each construct is used. The specified variable (e.g. n) sequentially takes the value of each element in the list. Note: The iterating variable contains a copy of the value. Modifying n within the loop does not modify the original list.

put (1, 2, 3) in Numbers
for each n in Numbers you do (
  put n * 2 in Double
  write Double
  newline
)
# Output:
# :) 2
# :) 4
# :) 6

Control Flow

The if construct

Conditional control structures allow executing blocks of code only when certain conditions occur. The fundamental instruction is if. The syntax is: if (CONDITION) then ( ... ) otherwise ( ... ).

For conditions, comparison operators are used: equals, is, >, <, >=, <=, !=.

Consider the following example, which checks driving license eligibility:

clear screen
get Age saying "How old are you?"
if (Age >= 18) then (
  write "You can get a driver's license."
) otherwise (
  write "You have to wait " + (18 - Age) + " more years."
)

Loops

Loops allow iterating a block of instructions multiple times, optimizing the code writing.

Definite Iteration

To execute instructions a predetermined number of times, the index loop is used. A local counter variable (often named i) is created and incremented automatically.

Syntax: index i goes from START to END and you do ( ... )

index i goes from 1 to 5 and you do (
  write "Iteration number " + i
  newline
)

Output:

:) Iteration number 1
:) Iteration number 2
:) Iteration number 3
:) Iteration number 4
:) Iteration number 5

Indefinite Iteration

This loop runs as long as the specified condition remains true. It is suitable when the number of iterations is not known in advance.

Syntax: as long as (CONDITION) you do ( ... )

Countdown example:

clear screen
put 10 in countdown
as long as (countdown > 0) you do (
  write countdown
  newline
  put countdown - 1 in countdown
  wait 1000 ms
)
write "Start."

The program decrements the variable every second and terminates when the condition countdown > 0 becomes false.

Stopping execution: stop

The stop command immediately halts the execution of the entire program. It is often used within a conditional structure to terminate a loop or game when a certain condition is met.

if (LivesLeft equals 0) then (
  write "Game Over"
  stop
)

Time Management

Waiting for time: wait

The execution of instructions is sequential and immediate. However, in contexts like animations, it is necessary to introduce pauses. The wait command suspends execution for a given interval. The unit of measurement is the millisecond (ms). Example:

wait 45 ms

This instruction forces the processor to pause for 45 milliseconds before moving to the next instruction.

Stopwatch: tick and tock

To measure elapsed time, Matita provides an integrated stopwatch.

  • tick: This is a command that starts or resets the stopwatch to the current moment.
  • tock: This is a special variable that contains the number of milliseconds elapsed since the last tick command.

Measurement example:

write "Measurement started..."
newline
tick # Start the stopwatch
wait 1500 ms
write "Elapsed time: " + tock + " ms"

Note: If you attempt to read the value of tock without having first executed tick, the program will report an error.

Text and Number Manipulation

Matita offers simple commands to analyze and transform strings and numbers by treating them as lists of characters or digits.

Transforming a string into a list

The decomposition command divides a string or a number into its individual characters, returning a list.

put "Hello" in Text
put decomposition(Text) in CharacterList
write CharacterList
# Output: :) (H, e, l, l, o)

Splitting by separator: split

Unlike simple character splitting, the split(text, separator) command divides a string based on a specific character (like a space or a comma).

put "red,green,blue" in CSV
put split(CSV, ",") in Colors
write Colors
# Output: :) (red, green, blue)

Specific Decomposition

The decomposition(value, TYPE) command offers greater control. The available target types are: DIGITS, CHARACTERS, and WORDS.

# Divide a sentence into words
put "Matita is fun" in Sentence
put decomposition(Sentence, "WORDS") in WordList
write WordList
# Output: :) (Matita, is, fun)

# Divide a number into digits
put 123.45 in Number
put decomposition(Number, "DIGITS") in DigitList
write DigitList
# Output: :) (1, 2, 3, ., 4, 5)

Composition

The inverse operation to decomposition is composition(list, TYPE), which joins the elements of a list into a single value. The available target types are TEXT and NUMBER.

put ("S", "e", "a") in Letters
put composition(Letters, "TEXT") in Word
write Word
# Output: :) Sea

# Create a number from digits
put (3, ".", 1, 4) in Digits
put composition(Digits, "NUMBER") in Value
write Value * 2 # We can do calculations because it's a number
# Output: :) 6.28

Type Inspection

In Matita, variables can change their type dynamically. It is possible to verify the nature of a data item using the following functions:

  • type(x): Returns a string describing the type of data (e.g., "number", "string", "list", "point", "boolean").
  • numeric(x) (or the equivalent is_number(x)): Returns true if the argument is a valid, finite number, false otherwise.
put 42 in X
write type(X)        # Output: number
write numeric(X)    # Output: true

put "Hello" in Y
write numeric(Y)    # Output: false

Type Conversion (Casting)

Sometimes it is necessary to treat a number as if it were text, for instance to concatenate it with other words, or conversely, to use it in a mathematical calculation. The cast command performs the type conversion operation.

  • cast(NUMBER): Returns a string containing the number.
  • cast(STRING): Returns a number (supports positive and negative integers, decimals, and scientific notation).

Example:

clear screen
# Test String to Number
put "10" in S1
put "3.14" in S2
put "-50" in S3
put "1.2e3" in S4  # 1200 in scientific notation

put cast(S1, "number") + cast(S2, "number") + cast(S3, "number") + cast(S4, "number") in Total
write "Sum of casts: " + Total 
newline

# Test Number to String (Concatenation)
put 500 in N1
put 12.5 in N2
# Sums the strings (concatenation) instead of the numbers
write "Concatenation: " + cast(N1, "text") + cast(N2, "text")

# Expected output:
# :) Sum of casts: 1163.14
# :) Concatenation: 50012.5

Unicode Function

Unicode is an international standard that assigns a unique numerical code to every character or symbol used in computing systems, including letters of all alphabets and Emojis.

The unicode(x) command acts as a bridge between numbers and symbols:

  • From Number to Symbol: If the argument is a number, it returns the associated character.
  • From Symbol to Number: If the argument is a string, it returns the numeric code of the first character.

Example:

# Example 1: Generate a symbol from the code (Rocket)
put unicode(128640) in RocketSymbol
write "To infinity... " + RocketSymbol
# Output: :) To infinity... 🚀
newline

# Example 2: Get the code from a symbol (Smiley)
put unicode("🙂") in SmileyCode
write "The code of the smiley is: " + SmileyCode
# Output: :) The code of the smiley is: 128578

Procedures and Functions

Using loops allows solving many problems. However, as complexity increases, it becomes necessary to organize the code in a modular way to promote readability and maintainability.

Matita supports procedural programming allowing the definition of custom commands. To avoid code duplication and encapsulate logic, Procedures and Functions are used:

  • A Procedure is a block of instructions that performs a well-defined task. Instructions that can be used in multiple points of the code are grouped into procedures. Procedures can apply the same operations on different parameters, provided as input. A procedure executes instructions but does not produce an output value.
  • A Function performs operations and returns a value (output) usable in other expressions via the return command.

Important: The Matita interpreter reads and executes the code line by line. Therefore, it is crucial that every procedure or function is defined before the line where the first call occurs.

Procedures

Example of defining and calling a procedure:

# Definition
define procedure greet(name) (
  write "Hello, " + name + ". Welcome to Matita."
  newline
)

# Call
greet("Alex")
greet("Maria")

Functions

Example of defining and calling a function:

define function Double(number) (
  put number * 2 in result
  return result
)

put 7 in Number
put Double(Number) in Doubled
write "The double of " + Number + " is " + Doubled
newline
Identifying constructs and built-in functions. Some expressions are identifying: they recall a part or property of something that already exists (e.g., x of pointP, item 3 of myList, dimension of myList). Other expressions are generative and produce a new value (built-in functions), for example point(10,20), list(1,2,3), sin(angle), squareroot(9).

Recursion

Recursion is a programming technique in which a function calls itself. It is particularly effective for solving problems that can be broken down into identical but smaller subproblems. The classic example is the calculation of the factorial (𝑛!).

define function factorial(n) (
  # Base case: termination condition.
  if (n equals 0) then (
    return 1
  ) otherwise (
    # Recursive step
    return n * factorial(n - 1)
  )
)

put 5 in Number
put factorial(Number) in Result
write Number + "! = " + Result # Will show 120

If the chain of recursive calls is excessive, a stack overflow will occur (recursion limit exceeded).

Memory Management

Variable Scope

The scope is the visibility area of a variable. In nested code blocks (inside you do, as long as, if, for each, etc.), the variables created with put are local to the block itself.

Variables defined in the main flow are called global. A global variable is readable within a local block, but if you try to assign a value to a variable with the same name in the local block, a new local variable is created that "hides" the global one (shadowing). In these cases, Matita issues a warning:

:) Warning: Local variable 'Number' is shadowing a global variable with the same name.

Consider the following example:

clear screen
put 1 in Number
as long as (Number < 6) you do (
  put Number + 1 in Number
  write Number
  newline
)
# now we are out of the local block
write "Final value: " + Number

Output:

:) Warning: Local variable 'Number' is shadowing a global variable with the same name.
:) 2
:) 3
:) 4
:) 5
:) 6
:) Final value: 1

As evident, the variable Number inside the loop is distinct from the external one. At the end of the loop, the global variable maintains its original value, which is 1.

The global directive

To modify a global variable within a local scope, it is necessary to declare it explicitly with the global directive:

clear screen
put 1 in Number
as long as (Number < 6) you do (
  global Number
  put Number + 1 in Number
  write Number
  newline
)
# now we are out of the block
write "Final value: " + Number

Output:

:) 2
:) 3
:) 4
:) 5
:) 6
:) Final value: 6

It is possible to declare multiple global variables at the same time by separating them with a comma:

global Name1, Name2 # multiple variables separated by commas

Value Semantics

The Matita interpreter adopts, by default, Value Semantics. Every assignment operation using the put command implies a Deep Copy of the data.

This means that assigning a list or a complex object to a new variable creates an independent and distinct clone in memory.

put (1, 2, 3) in A
put A in B      # B is a deep copy of A
put 99 in item 1 of B
# A remains (1, 2, 3), B becomes (99, 2, 3)

This design choice makes data management safe and predictable.

Access by Reference: alias

In several languages, including Python and Java, the instruction B = A makes both variables point to the same object in memory. It is said that such languages implicitly adopt a reference semantics and A and B are aliases, that is, replicas of each other. An alias therefore contains no data of its own, but acts as a pointer towards another target variable.

For clarity, Matita adopts value semantics by default, similarly to C and C++. In Matita the instruction put A in B duplicates the value of A in B, performing a deep copy of A. The result will be that two distinct variables will exist, but with the same initial content.

To create a shared reference, for example to allow data sharing or resource optimization, Matita introduces the concept of explicit aliasing.

An alias is created via explicit request using the command: B alias A, where A is an existing variable.

Any read or write operation performed on B is intercepted by the interpreter and redirected to the target variable A. The following code:

put 4 in A
put 56 in (B alias A)
write "A: " + A + "; B: " + B

produces as output:

:) A: 56; B: 56

Passing by Reference to Procedures: inout

Matita supports two modes of passing parameters to a procedure.

  1. Pass by value (default): The value of the argument is copied into the local scope of the procedure. Changes do not propagate to the caller.
  2. Pass by reference (via the inout keyword): By using the inout keyword, the interpreter creates a local alias that points to the variable passed by the caller.
define procedure Increment(inout X) (
    put X + 1 in X  # Directly modifies the external variable
)

Freeing memory: null

The null keyword indicates a special value corresponding to the absence of data. This keyword allows emptying a variable.

For example, writing put null in VAR disconnects the name VAR from its previous content, which signals the computer that the content is no longer needed and the memory it occupied can be freed.

Using null is useful for freeing computer memory when heavy objects, such as images or very long lists, are no longer necessary.

Part Two: Graphics with Matita

Graphic Elements

The graphics area uses a Cartesian coordinate system. The position of each element is defined by a pair of coordinates (x, y). The origin (0, 0) is located at the center of the area. Positive values of x indicate positions to the right, negative values to the left. Positive values of y indicate positions upwards, negative values downwards.

Coordinates are represented as lists of two numbers. The explicit syntax point(x, y) can be used. To access the individual components of a point, the x of and y of properties are used.

put point(10, 44) in MyPoint
put x of MyPoint in XCoordinate
write XCoordinate

To modify a single coordinate, the set command is used. This operation is fundamental to manage the movement of objects.

# Move the point 10 steps down without changing the x
set y of MyPoint to (y of MyPoint - 10)
write y of MyPoint # Will show 34

Colors and Backgrounds

Matita uses the RGB color model. Colors are defined by a list of three values (R, G, B), representing respectively the percentages of Red, Green, and Blue, ranging from 0 to 100, inclusive.

Example setting the background:

put (68, 85, 90) in skyBlue
set color skyBlue
paint background

To erase the contents of the graphic area, for example before drawing a new frame in an animation, the clear graphics command is used.

Drawing Basic Shapes

After setting the current color, it is possible to draw graphic primitives.

Circle: Syntax circle radius R at POS. Draws a filled disk.

set color (100, 87, 0)
circle radius 60 at (120, 80)

Circumference: Syntax circumference radius R thickness S at POS. Draws only the outline.

set color (0, 0, 0)
circumference radius 25 thickness 4 at (-200, -50)

Line: Syntax line thickness S from P1 to P2. Draws a segment between two points.

set color (0, 0, 0)
line thickness 2 from (-200, -75) to (-200, -160)

Triangle: Syntax triangle with vertices P1, P2, P3. Draws a filled triangle connecting the three specified points separated by commas.

set color (0, 50, 100)
triangle with vertices (0, 50), (-50, -50), (50, -50)

Arc: Syntax arc radius R thickness S from P1 to P2. Draws a circular arc of a given radius, passing through the two given points. Since there are at most two circumferences of a given radius passing through two assigned points in the plane, the code will draw the arc that goes from the first to the second point rotating counterclockwise. This implies that if you reverse the order of the two points, you will get an arc with inverted concavity.

set color (100, 0, 0)
arc radius 240 thickness 12 from (-220, -160) to (220, -160)

Composite example:

clear graphics

put 15 in S       # Thickness of each arc
put 260 in R      # Radius of the outermost color (Red)
put -160 in Y     # Horizon height (where the grass ends)

# --- SKY AND BACKGROUND ---
set color (68, 85, 90)
paint background

# --- GRASS ---
set color (16, 65, 27)
rectangle from (-360, -250) to (360, Y)

# --- SUN ---
set color (100, 87, 0)
circle radius 70 at (250, 180)

# --- RAINBOW ---
# 1. Red
set color (90, 0, 0)
arc radius R thickness S from (R, Y) to (-R, Y)
put R - S in R  # Shrink the radius for the next color

# 2. Orange
set color (90, 50, 0)
arc radius R thickness S from (R, Y) to (-R, Y)
put R - S in R

# 3. Yellow
set color (90, 90, 0)
arc radius R thickness S from (R, Y) to (-R, Y)
put R - S in R

# 4. Green
set color (0, 70, 0)
arc radius R thickness S from (R, Y) to (-R, Y)
put R - S in R

# 5. Blue
set color (0, 0, 90)
arc radius R thickness S from (R, Y) to (-R, Y)
put R - S in R

# 6. Indigo / Violet
set color (40, 0, 70)
arc radius R thickness S from (R, Y) to (-R, Y)

# --- BALLOON ---
set color (80, 0, 90) # A brighter purple
circle radius 25 at (-100, -50)
line thickness 2 from (-100, -75) to (-100, Y)

Graffiti (Text)

The graffiti command allows inserting text directly into the graphic area.

set color (10, 10, 10)
graffiti size 30 at point(0, 0) "Center."

Redrawing a Scene

The redraw command forces an update of the graphic area (canvas). This command is essential for displaying changes that have occurred in the scene stored in memory.

Framing the Graphics (Auto-framing)

The frame graphics command automatically adjusts the camera (pan and zoom) so that all objects currently present in the graphic area (canvas) are visible and centered in the window.

This command is useful for making the scene dynamically fit the screen. It also implicitly performs a redraw, so using it makes redraw unnecessary.

set color (100, 0, 0)
# Draws a very large circle, far from the origin
circle radius 500 at (2000, 2000)
# Adjust the view to show it all
frame graphics

Creating Animations

To achieve the movement and animation of objects in Matita, the principle of persistence of vision is used: a series of static images, slightly different from each other, shown in rapid succession, creates the illusion of movement.

The typical structure of an animation loop involves the following steps:

  1. Start a loop (usually as long as (true)).
  2. Clear the previous graphics (with clear graphics or paint background).
  3. Update the coordinates of the objects (for example, by moving a point).
  4. Draw the objects in the new positions.
  5. Force the graphical update with redraw (optional if using wait, which does it implicitly).
  6. Wait a short time interval (e.g., wait 16 ms for approximately 60 frames per second).

Simple animation example:

put -150 in X
as long as (X < 150) you do (
  put X + 2 in X
  clear graphics
  circle radius 20 at point(X, 0)
  wait 16 ms
  redraw
)

Keyboard Input

To achieve real-time interactions, the state of the keys is checked. The expression key "KEY_NAME" is pressed returns true if the indicated key is currently being pressed. The directional keys are mapped as UP, DOWN, LEFT, RIGHT.

Other control keys: SPACE, ENTER, ESC, TAB, BACKSPACE, SHIFT, CTRL, ALT. Alphanumeric characters are simply indicated with their symbol ("A", "x", etc.).

Example of controlling an object via keyboard:

put point(0,0) in playerPos
put 300 in lim

as long as (true) you do (

  if (key "UP" is pressed) then (
    set y of playerPos to (y of playerPos + 5)
  )
  if (key "DOWN" is pressed) then (
    set y of playerPos to (y of playerPos - 5)
  )
  if (key "RIGHT" is pressed) then (
    set x of playerPos to (x of playerPos + 5)
  )
  if (key "LEFT" is pressed) then (
    set x of playerPos to (x of playerPos - 5)
  )

  # Emergency exit
  if (key "x" is pressed) then (stop)

  # Drawing
  clear graphics
  set color (0, 100, 100)
  rectangle from (-lim, -lim) to (lim, lim)

  set color (0, 50, 100)
  circle radius 20 at playerPos

  redraw

  # Refresh rate (around 60 fps)
  wait 16 ms
)

Graphic Labels

To optimize the code and reuse groups of shapes, you can define labels. A label groups a series of drawing instructions into a single reusable object. Syntax: label (...) as NAME.

Example of definition and usage:

# Model definition
label (
  # Body
  set color (100, 100, 100) # White
  circle radius 30 at (0, 0)
  circle radius 20 at (0, 45)
  circle radius 15 at (0, 75)

  # Details
  set color (0, 0, 0) # Black
  circle radius 2 at (-5, 78)
  circle radius 2 at (5, 78)
  circle radius 2 at (0, 50)
  circle radius 2 at (0, 40)

  # Nose
  set color (100, 60, 0) # Orange
  triangle with vertices (0, 75), (0, 73), (8, 74)
) as Snowman

set color (50,50,50)
paint background
draw Snowman at point(150, 0)
draw Snowman at point(-100, 0)

Transformations: Scaling and Rotation

Labeled objects can undergo geometric transformations.

Scaling

The scale command resizes the label. A factor of 2 doubles the dimensions, 0.5 halves them.

Note: The scaling operation is permanent for the label to which it is applied.

scale Snowman by 0.5
draw Snowman at point(0, 100)
draw Snowman at point(0, -100)

Rotation

It is possible to rotate an object with respect to the center of its bounding box. The angle is expressed in sexagesimal degrees (0-360), with counterclockwise rotation for positive values.

rotate Snowman of 45 deg and place it at (300, -300)

Images

External image files can be imported for use as graphic objects or backgrounds. The files must reside in the same directory as the matita.html file.

Images as Labels

An imported image is treated as a label and can therefore be drawn, scaled, and rotated.

import "Filename.png" as kitten
draw kitten at (0, 0)
scale kitten by 0.5
rotate kitten of 15 deg and place it at (-120, -60)

Images as Backgrounds

To set an image as the background of the entire graphic area:

load "grass.png" scale 0.5 as background
paint background

Importing External Code (Libraries)

It is possible to split complex programs across multiple files. By using the import command without the as keyword, Matita will read a text file containing source code (e.g., useful procedures or functions) and execute it as if it were written directly in the main file.

import "my_library.txt"
# Now you can use the functions defined in my_library.txt

Interacting with the Canvas

Besides rendering the graphics generated by the code, Matita's graphics area (the canvas) offers interactive tools to inspect the drawing space and locate points.

Pan and Zoom

You can navigate inside the graphic area without altering the original coordinates of the objects:

  • Pan (Dragging): By holding down the right mouse button (or the middle button, or Shift + left click) and moving the cursor, it's possible to translate the view in any direction.
  • Zoom: Using the mouse wheel, you enlarge or shrink the view. The magnification is centered on the cursor, pointing towards the coordinates where the mouse is at that moment.

Orthogonal Constraints (Ortho) and Magnet (Snap)

The buttons located on the top bar of the canvas window allow activating geometric precision tools:

  • Snap: If active, the cursor is attracted and hooked to notable points of the objects already drawn (circle centers, rectangle vertices, line midpoints, etc.).
  • Ortho (⟂) and Alpha Angle (α): By activating the Ortho tool, you can draw lines to support alignment. With a single click on the canvas, a base point is fixed; moving the mouse, the crosshair will be constrained to move only horizontally or vertically. By clicking the α button and entering a numeric value in the adjacent field, you can freely rotate the orthogonal axes by that number of degrees. A second click releases the base point.
  • Save PNG: Instantly exports and downloads the current content of the canvas as a .png image.
  • Fit to Window (⛶): Automatically adjusts the zoom and camera position to perfectly frame all drawn objects within the graphic window.

Coordinate Transfer

To speed up coding and object positioning, Matita allows picking up positional data directly from the graphic area and inserting them into the text editor.

Performing a long click with the left button on a point in the canvas, the coordinates of that point (for example, (15.2, 30.5)) will be automatically typed into the code Editor at the text cursor. If this operation is performed while the Snap is active and hooked to a point whose coordinates are saved in a global variable, the name of that variable will be pasted directly into the Editor instead of the numerical values, preserving code consistency.

Guided Exercise: Interactive Smiley

The following code integrates label definitions, user input, and a rendering loop to create a controllable object.

# Define the colors
put (100, 100, 100) in White
put (0, 0, 0) in Black
put (100, 90, 10) in Yellow

# Dimensions of the white background 
put 300 in lim

# Define the Smiley as a graphic object (label)
label (
  set color Yellow
  circle radius 120 at point(0, 0)
  set color Black
  circumference radius 120 thickness 8 at point(0, 0)
  circle radius 12 at point(-40, 30)
  circle radius 12 at point(40, 30)
  arc radius 60 thickness 8 from (-50, -25) to (50, -25)
) as Smiley

# Variable Initialization
put point(0, 0) in Position
put 0 in Angle

clear screen
write "Controls: Arrow Keys (Move), A/D (Rotate), X (Exit)"
newline

# Main game loop
as long as (true) you do (
  # Global variables we modify in the loop
  global Position, Angle

  if (key "x" is pressed) then (stop)

  # Movement
  if (key "UP" is pressed) then (
    set y of Position to (y of Position + 5)
  )
  if (key "DOWN" is pressed) then (
    set y of Position to (y of Position - 5)
  )
  if (key "LEFT" is pressed) then (
    set x of Position to (x of Position - 5)
  )
  if (key "RIGHT" is pressed) then (
    set x of Position to (x of Position + 5)
  )

  # Rotation
  if (key "a" is pressed) then (
    put Angle + 5 in Angle
  )
  if (key "d" is pressed) then (
    put Angle - 5 in Angle
  )

  # Drawing the current frame
  clear graphics
  set color White
  rectangle from (-lim, -lim) to (lim, lim)
  rotate Smiley of Angle deg and place it at Position
  
  redraw
  wait 16 ms
)
