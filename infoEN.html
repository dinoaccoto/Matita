The Matita Language

Matita (Micro Accessible Tinkering Language for Instant Type-and-Play Applications) is an open-source language designed to reduce the initial friction when learning to program.

The language aims to make source code as readable as pseudocode: the commands are close to being complete sentences in natural language (e.g., output "hello", put 45 in A, as long as (...) you do), allowing the student to focus on algorithmic logic rather than syntax.

A distinctive feature is its native bilingualism: every Italian instruction has an English counterpart (e.g., if/se, as long as/fintanto) that can be used simultaneously within the same file. This allows a gradual transition towards standard technical English without changing the development environment.

A central aspect of Matita's philosophy is immediate access to graphics. The use of geometric primitives does not require importing external libraries or setting up a graphical environment: commands like circle or line are as native as addition. This transforms the language into a tool for immediate geometric visualization.

The code is interpreted line by line, with dynamic typing and immediate feedback, favoring a procedural style and rapid prototyping. Although the language is Turing-complete, the interpreter includes loop protection mechanisms and recursion limits to prevent browser freezing in case of logical errors.

The Development Environment (IDE)

The reference environment, Matita Runtime & IDE, is technically a Single Page Application (SPA) contained within a single HTML file. The absence of dependencies on specific operating systems guarantees total portability: the matita.html file can be distributed via a USB flash drive or downloaded and used on any computer.

The interface is divided into three resizable areas:

  1. Editor: with syntax highlighting and contextual auto-completion.
  2. Canvas: a vector graphics area that automatically manages scaling and allows PNG exporting.
  3. Terminal: for textual I/O and debugging.

Design and Comparison with Other Languages

Matita is inspired by the philosophy of BASIC for its immediacy, but it incorporates concepts drawn from other languages to remain modern and useful in facilitating the transition towards industrial languages.

  • Program Structure: To promote methodological rigor, Matita explicitly distinguishes between procedures (which execute actions) and functions (which return values). This separation, inspired by Pascal, helps students understand the semantic difference between executing a task and calculating a result. The parameters of a procedure can also be passed by reference.
  • Centrality of Lists: The list is the fundamental data structure in Matita. It is used not only for ordered collections of homogeneous or heterogeneous data, but also for (x,y) point coordinates and (r,g,b) colors. Indexing starts at 1 (as in MATLAB or Julia), a choice considered more natural compared to the 0-based indexing found in many C-derived languages.
  • Data and Memory Management:
    • Value Semantics: The command put A in B creates an independent copy of the data, increasing code predictability. The syntax emphasizes the validity of the metaphor that a variable is a box with a label (its name) and a content (its value). Aliasing, which involves shared references to the same memory cell, is possible but must be intentional using the alias command.
    • Casting and Null: Types can be explicitly converted via casting commands (e.g., from string to number). Furthermore, assigning null to a variable allows resources to be freed up, introducing the concept of memory cleanup but without the complexity of pointers.

In conclusion, Matita is a serious toy that provides a protected environment ("sandbox") where one can experiment with algorithmic thinking, mathematics, logic, and geometry, in preparation for the complexity of industrial programming languages.
