4. Caesar Cipher

Implements an ancient cryptographic algorithm with simple substitution.
Shifts each letter of the text by a fixed number of positions, determined by the key.
Uses the modulo operator (mod) to handle wrapping the alphabet from Z to A.

Matita 
get text saying "Enter the text to encrypt (in lowercase):"
get shiftValue saying "Enter the key (integer):"

put split(text) in charList
put unicode("a") in baseA

clear screen
write "Original Text: " + text; newline
write "Ciphertext:    "

for each letter in charList you do (
    put unicode(letter) in code
    if (code >= baseA and code <= baseA + 25) then (
        put code - baseA in index0
        put index0 + shiftValue in shiftedIndex
        put shiftedIndex mod 26 in newIndex
        put newIndex + baseA in cipherCode
        write unicode(cipherCode)
    ) otherwise (# Leave unchanged if not a letter (space, punctuation, etc.):
        write letter
    )
)

8. Guess the Number

An example of textual input/write and complex conditional logic.
The computer uses binary search to guess the number the user is thinking of in just a few steps.
Processes user responses (Greater, Smaller, Correct) to narrow down the range.

Matita 
# The computer guesses the number (1..100) the user has in mind.

clear screen
get start saying "Think of an integer between 1 and 100.
I will try to guess it as quickly as possible.
Answer with:
G = your number is Greater
S = your number is Smaller
C = Correct

X = exit
Press ENTER to start..."

put 1 in Min
put 100 in Max
put 0 in Attempts

as long as (true) you do (
    global Min, Max, Attempts

    put trunc((Min + Max) / 2) in Guess
    put Attempts + 1 in Attempts

    get Answer saying ("Guess: " + Guess + ". Is the number you thought of Greater (press G), Smaller (press S), or is it Correct (press C)? Press X to exit: ")

    if (Answer equals "X") or (Answer equals "x") then (
        write "Attempts made: " + Attempts; newline
        stop
    )

    if (Answer equals "C") or (Answer equals "c") then (
        newline
        write "Guessed! The number was " + Guess + "."
        newline
        write "Attempts: " + Attempts; newline
        stop
    )

    if (Answer equals "G") or (Answer equals "g") then (
        put Guess + 1 in Min
    ) otherwise (
        if (Answer equals "S") or (Answer equals "s") then (
            put Guess - 1 in Max
        ) otherwise (
            newline; write "Invalid answer. Use G, S, C, or X."
            newline
        )
    )

    if (Min > Max) then (
        newline
        write "Inconsistency in answers: no compatible number."
        newline
        write "Attempts made: " + Attempts; newline
        stop
    )
)

9. Fractal Tree

An example of recursion to draw a natural-looking tree.
Includes mathematical functions (sine, cosine) to calculate the branches.
Uses random numbers to vary angles and lengths, making each tree unique.

Matita 
# Fractal Tree
clear screen
clear graphics

# Rectangular leaf anchored at the bottom "stem"
define procedure drawLeaf(center, baseAngle) (
    set color list(random(30, 90), random(80, 100), random(10, 50))
    
    put random(5, 9) in leafWidth
    put random(6, 12) in leafHeight
    
    # Total rotation in radians
    put (baseAngle + random(-20, 20)) * pi / 180 in angleRad
    put cos(angleRad) in c
    put sin(angleRad) in s

    # Local rectangle anchored at the bottom
    put -leafWidth / 2 in x1_loc
    put 0 in y1_loc
    put leafWidth / 2 in x2_loc
    put leafHeight in y2_loc

    # Four vertices after rotation + translation
    put (x1_loc * c - y1_loc * s + x of center) in v1x
    put (x1_loc * s + y1_loc * c + y of center) in v1y
    put (x2_loc * c - y1_loc * s + x of center) in v2x
    put (x2_loc * s + y1_loc * c + y of center) in v2y
    put (x2_loc * c - y2_loc * s + x of center) in v3x
    put (x2_loc * s + y2_loc * c + y of center) in v3y
    put (x1_loc * c - y2_loc * s + x of center) in v4x
    put (x1_loc * s + y2_loc * c + y of center) in v4y

    triangle with vertices point(v1x, v1y), point(v2x, v2y), point(v3x, v3y)
    triangle with vertices point(v1x, v1y), point(v4x, v4y), point(v3x, v3y)
    
    # Short stem aligned with the leaf
    set color list(50, 30, 15)
    put (0 * c - (-2) * s + x of center) in s2x
    put (0 * s + (-2) * c + y of center) in s2y
    line thickness 1 from center to point(s2x, s2y)
)

# Recursive branch; leaves grow at the tip
define procedure drawBranch(startPos, branchLength, angle, level) (
    if (level > 0) then (
        put (angle * pi / 180) in angleRad
        put (x of startPos + branchLength * cos(angleRad)) in endX
        put (y of startPos + branchLength * sin(angleRad)) in endY
        put point(endX, endY) in endPos

        set color list(40, 26, 13)
        line thickness max(1, level / 2) from startPos to endPos
        
        put (0.75 + random(-10, 5) / 100) in lengthFactor
        put random(-15, 15) in var1
        put random(-15, 15) in var2
        
        drawBranch(endPos, branchLength * lengthFactor, angle - (25 + var1), level - 1)
        drawBranch(endPos, branchLength * lengthFactor, angle + (25 + var2), level - 1)
    )
    
    if (level == 0) then (
        # Leaf point moved forward along the branch and slightly to the side
        put (angle * pi / 180) in a
        put cos(a) in ca
        put sin(a) in sa
        put random(6, 12) in forward              # offset along the branch
        put random(-3, 3) in sideways               # lateral offset
        # perpendicular to the branch: (-sa, ca)
        put (x of startPos + forward * ca + sideways * (-sa)) in fx
        put (y of startPos + forward * sa + sideways * ca) in fy
        put point(fx, fy) in p_leaf
        
        # 2-4 leaves per tip, with small variations
        put random(2, 4) in n
        index i goes from 1 to n and you do (
            put angle + random(-15, 15) in a_loc
            # micro-shift to prevent overlapping
            put fx + random(-2, 2) in lx
            put fy + random(-2, 2) in ly
            drawLeaf(point(lx, ly), a_loc)
        )
    )
)

# Start
write "Drawing a fractal tree with natural leaves..."
put point(0, -220) in startPoint
drawBranch(startPoint, 90, 90, 9)

frame graphics
write "Done! Run again for variation."

10. Collect and Escape

A complete video game with a basic physics engine.
Control the blue circle with the arrow keys to collect food and avoid enemies.
Implements vectors, velocity, inertial acceleration, and collision detection.

Matita 
# Collect and Escape
# Move the blue circle.
# Collect the energy particles (light blue squares).
# If a triangle hits you, you lose a life.
# You start with 3 lives. Press 'X' to exit.

# --- Initial Settings ---

# Global state variables
put 0 in X_POS
put 0 in Y_POS
put 5 in SPEED
put 0 in EXIT_GAME 
put 0 in Score
put 0 in FOOD_X
put 0 in FOOD_Y
put 3 in LIVES
put 0 in GAME_OVER

# Size constants
put 20 in PLAYER_RADIUS
put 18 in TRIANGLE_RADIUS # effective radius closer to drawn triangle
put 15 in FOOD_SIZE
put 220 in PLAY_AREA 

# --- Physics variables ---
put 10 in TRIANGLE_MASS
put 20 in K_SPRING
put 5 in C_DAMPER
put 50 in L_REST
put 0.030 in DT          # Delta Time (corresponds to "wait 30 ms")

# --- Lists for Triangles ---
put list() in LIST_TRIANGLE_X
put list() in LIST_TRIANGLE_Y
put list() in LIST_TRIANGLE_VX
put list() in LIST_TRIANGLE_VY

# Set background color (white)
set color (100, 100, 100)
paint background

# --- Define Procedures ---

define procedure place_food () (
    global FOOD_X, FOOD_Y, PLAY_AREA
    put random(-PLAY_AREA, PLAY_AREA) in FOOD_X
    put random(-PLAY_AREA, PLAY_AREA) in FOOD_Y
)

# Adds a triangle that doesn't overlap the player
define procedure add_triangle () (
    global LIST_TRIANGLE_X, LIST_TRIANGLE_Y, LIST_TRIANGLE_VX, LIST_TRIANGLE_VY
    global PLAY_AREA, X_POS, Y_POS

    put random(-PLAY_AREA, PLAY_AREA) in TX
    put random(-PLAY_AREA, PLAY_AREA) in TY

    put TX - X_POS in DX
    put TY - Y_POS in DY
    put squareroot((DX*DX) + (DY*DY)) in D

    if (D < 120) then (
        if (abs(DX) > abs(DY)) then (
            if (DX >= 0) then ( put PLAY_AREA in TX ) otherwise ( put -PLAY_AREA in TX )
        ) otherwise (
            if (DY >= 0) then ( put PLAY_AREA in TY ) otherwise ( put -PLAY_AREA in TY )
        )
    )

    add TX to LIST_TRIANGLE_X
    add TY to LIST_TRIANGLE_Y
    add 0 to LIST_TRIANGLE_VX
    add 0 to LIST_TRIANGLE_VY

    write "A new red triangle has appeared!"
    newline
)

define procedure update_game () (
    global X_POS, Y_POS, EXIT_GAME, SPEED, LIVES, GAME_OVER
    global Score, FOOD_X, FOOD_Y, FOOD_SIZE, PLAYER_RADIUS, TRIANGLE_RADIUS, PLAY_AREA
    global TRIANGLE_MASS, K_SPRING, C_DAMPER, L_REST, DT
    global LIST_TRIANGLE_X, LIST_TRIANGLE_Y, LIST_TRIANGLE_VX, LIST_TRIANGLE_VY

    # 1. Player movement
    if (key "UP" is pressed) then ( put Y_POS + SPEED in Y_POS )
    if (key "DOWN" is pressed) then ( put Y_POS - SPEED in Y_POS )
    if (key "LEFT" is pressed) then ( put X_POS - SPEED in X_POS )
    if (key "RIGHT" is pressed) then ( put X_POS + SPEED in X_POS )
    if (key "X" is pressed) then ( put 1 in EXIT_GAME )

    # --- 2. PHYSICS and TRIANGLE COLLISIONS ---
    put dimension of LIST_TRIANGLE_X in NUM_TRIANGLES

    if (NUM_TRIANGLES > 0) then (
        index i goes from 1 to NUM_TRIANGLES and you do (
            put item i of LIST_TRIANGLE_X in T_X
            put item i of LIST_TRIANGLE_Y in T_Y
            put item i of LIST_TRIANGLE_VX in T_VX
            put item i of LIST_TRIANGLE_VY in T_VY

            put X_POS - T_X in VECTOR_X
            put Y_POS - T_Y in VECTOR_Y
            put squareroot((VECTOR_X * VECTOR_X) + (VECTOR_Y * VECTOR_Y)) in DISTANCE
            put 0 in EXTENSION
            if (DISTANCE > 0.1) then ( put DISTANCE - L_REST in EXTENSION )
            put K_SPRING * EXTENSION in SPRING_FORCE_MAG

            put 0 in DIR_X
            put 0 in DIR_Y
            if (DISTANCE > 0) then (
                put VECTOR_X / DISTANCE in DIR_X
                put VECTOR_Y / DISTANCE in DIR_Y
            )

            put (SPRING_FORCE_MAG * DIR_X) - (C_DAMPER * T_VX) in TOT_FORCE_X
            put (SPRING_FORCE_MAG * DIR_Y) - (C_DAMPER * T_VY) in TOT_FORCE_Y
            put TOT_FORCE_X / TRIANGLE_MASS in ACCEL_X
            put TOT_FORCE_Y / TRIANGLE_MASS in ACCEL_Y
            put T_VX + (ACCEL_X * DT) in T_VX_NEW
            put T_VY + (ACCEL_Y * DT) in T_VY_NEW
            put T_X + (T_VX_NEW * DT) in T_X_NEW
            put T_Y + (T_VY_NEW * DT) in T_Y_NEW

            put T_X_NEW in item i of LIST_TRIANGLE_X
            put T_Y_NEW in item i of LIST_TRIANGLE_Y
            put T_VX_NEW in item i of LIST_TRIANGLE_VX
            put T_VY_NEW in item i of LIST_TRIANGLE_VY

            # --- Player-Triangle Collision ---
            put T_X_NEW - X_POS in DIST_TX
            put T_Y_NEW - Y_POS in DIST_TY
            put (DIST_TX * DIST_TX) + (DIST_TY * DIST_TY) in DIST_T_SQ
            put PLAYER_RADIUS + TRIANGLE_RADIUS in R_TOT_T
            put R_TOT_T * R_TOT_T in R_TOT_T_SQ

            if (DIST_T_SQ < R_TOT_T_SQ) then (
                put LIVES - 1 in LIVES
                if (LIVES <= 0) then (
                    put 1 in GAME_OVER
                ) otherwise (
                    write "LIFE LOST! Lives left: "
                    write LIVES
                    newline
                    put 0 in X_POS
                    put 0 in Y_POS
                    index j goes from 1 to NUM_TRIANGLES and you do (
                        put random(-PLAY_AREA, PLAY_AREA) in item j of LIST_TRIANGLE_X
                        put random(-PLAY_AREA, PLAY_AREA) in item j of LIST_TRIANGLE_Y
                        put 0 in item j of LIST_TRIANGLE_VX
                        put 0 in item j of LIST_TRIANGLE_VY
                    )
                    wait 500 ms
                )
            )
        )
    )

    # --- 3. Food Collision Logic ---
    put X_POS - FOOD_X in DIST_X
    put Y_POS - FOOD_Y in DIST_Y
    put (DIST_X * DIST_X) + (DIST_Y * DIST_Y) in DIST_SQ
    put PLAYER_RADIUS + (FOOD_SIZE / 2) in TOTAL_RADIUS
    put TOTAL_RADIUS * TOTAL_RADIUS in TOTAL_RADIUS_SQ

    if (DIST_SQ < TOTAL_RADIUS_SQ) then (
        put Score + 1 in Score
        write "Score: "
        write Score
        newline
        place_food()
        add_triangle()
    )

    # --- 4. Update graphics ---
    put dimension of LIST_TRIANGLE_X in NUM_TRIANGLES
    clear graphics
    set color (100, 100, 100)
    paint background

    # Draw Player (Blue) first
    set color (0, 0, 100)
    circle radius PLAYER_RADIUS at point(X_POS, Y_POS)

    # Draw Food (Light Blue)
    set color (50, 80, 100)
    put FOOD_X - (FOOD_SIZE / 2) in FOOD_X1
    put FOOD_Y - (FOOD_SIZE / 2) in FOOD_Y1
    put FOOD_X + (FOOD_SIZE / 2) in FOOD_X2
    put FOOD_Y + (FOOD_SIZE / 2) in FOOD_Y2
    rectangle from point(FOOD_X1, FOOD_Y1) to point(FOOD_X2, FOOD_Y2)

    # Draw Triangles and Springs above player
    if (NUM_TRIANGLES > 0) then (
        index i goes from 1 to NUM_TRIANGLES and you do (
            put item i of LIST_TRIANGLE_X in T_X
            put item i of LIST_TRIANGLE_Y in T_Y

            set color (50, 50, 50)
            line thickness 2 from point(X_POS, Y_POS) to point(T_X, T_Y)

            set color (100, 0, 0)
            triangle with vertices point(T_X, T_Y + 15), point(T_X - 10, T_Y - 10), point(T_X + 10, T_Y - 10)
        )
    )

    # HUD
    set color (0, 0, 0)
    graffiti size 16 at point(-PLAY_AREA - 20, PLAY_AREA + 20) "Score: " + Score
    graffiti size 16 at point(-PLAY_AREA - 20, PLAY_AREA) "Lives: " + LIVES

    redraw
)

# --- Main Game Loop ---
place_food()

as long as (EXIT_GAME equals 0 and GAME_OVER equals 0) you do (
    update_game()
    wait 30 ms 
)

# --- End Screen ---
if (GAME_OVER equals 1) then (
    clear graphics
    set color (100, 100, 100)
    paint background
    set color (100, 0, 0)
    graffiti size 50 at (-130, 20) "GAME OVER"
    set color (0, 0, 0)
    graffiti size 20 at (-90, -40) "Final Score: " + Score
    redraw
) otherwise (
    write "Game Ended! Final Score: "
    write Score
    newline
)
stop

11. Koch Snowflake

The Koch snowflake is one of the most famous fractal curves.
This program draws it using recursion: each straight line is divided into four segments.
The recursion depth determines the level of detail of the final fractal.

Matita 
# Program to draw the Koch snowflake fractal
#
# We define the level of detail for the fractal.
# A value of 4 or 5 is a good start.
# Higher values (6+) can be very slow.
put 6 in maxLevel

# We set line thickness and color
put 1 in lineThick
set color (0, 40, 80) # Dark blue

# We calculate the constants for the 60-degree rotation (PI/3 radians)
# The interpreter uses Math.sin/cos, which require radians.
put pi / 3 in PITHIRD
put cos(PITHIRD) in cos60
put -sin(PITHIRD) in sin60

#
# This is the recursive procedure that draws a Koch curve
# between two points (x1, y1) and (x2, y2).
#
define procedure drawKoch(level, x1, y1, x2, y2) (
    
    # Base case: if level is 0, draw a straight line.
    if (level == 0) then (
        line thickness lineThick from (x1, y1) to (x2, y2)
    )
    
    # Recursive case: calculate the 4 segments and repeat.
    otherwise (
        # 1. Calculate the vector from point 1 to point 2
        put x2 - x1 in dx
        put y2 - y1 in dy
        
        # 2. Find points A and B, dividing the line into 3 parts
        put x1 + dx / 3 in ax
        put y1 + dy / 3 in ay
        
        put x1 + dx * 2 / 3 in bx
        put y1 + dy * 2 / 3 in by
        
        # 3. Calculate point C (the "tip") by rotating segment A->B
        #    60 degrees around point A.
        put (dx / 3) * cos60 - (dy / 3) * sin60 in cx_rel
        put (dx / 3) * sin60 + (dy / 3) * cos60 in cy_rel
        
        put ax + cx_rel in cx
        put ay + cy_rel in cy
        
        # 4. Call the procedure 4 times for the new segments
        drawKoch(level - 1, x1, y1, ax, ay)  # P1 -> A
        drawKoch(level - 1, ax, ay, cx, cy)  # A -> C
        drawKoch(level - 1, cx, cy, bx, by)  # C -> B
        drawKoch(level - 1, bx, by, x2, y2)  # B -> P2
    )
)

# --- MAIN PROGRAM ---

# We define the 3 vertices of a large equilateral triangle
# centered at the origin (0, 0).
put 1000 in radius

# Point 1 (top)
put radius * cos(pi / 2) in p1x
put radius * sin(pi / 2) in p1y

# Point 2 (bottom left)
put radius * cos(7 * pi / 6) in p2x
put radius * sin(7 * pi / 6) in p2y

# Point 3 (bottom right)
put radius * cos(11 * pi / 6) in p3x
put radius * sin(11 * pi / 6) in p3y

# We draw the snowflake by calling the procedure
# for each of the 3 sides of the triangle.
drawKoch(maxLevel, p1x, p1y, p2x, p2y)
drawKoch(maxLevel, p2x, p2y, p3x, p3y)
drawKoch(maxLevel, p3x, p3y, p1x, p1y)

12. Polar Curves

Generation of complex geometric figures using polar coordinates.
Converts radius and angle to Cartesian coordinates (x, y) for drawing.
Uses mathematical loops to vary color and shape while drawing.

Matita 
put 300 in R0
put 5000 in steps
put 2*pi/steps in inc
