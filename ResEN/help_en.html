Programming in Matita

Part One: Programming Fundamentals

Language Basics

Programming is the activity that leads to creating an ordered set of instructions that a computer can execute to perform specific tasks. Unlike natural language, which can be ambiguous, communicating with a computer requires the use of a precise formal language.

By writing source code (or simply "code"), the programmer implements algorithms, which are logical procedures that transform input data into the desired output.

The Integrated Development Environment

An "Integrated Development Environment" (IDE) consists of all the tools necessary to write a program, execute it, debug it, and view the results.

Matita's user interface has three main parts:

  • Editor: The area dedicated to writing the source code.
  • Graphics Area (Canvas): The visual space where drawings and animations are rendered.
  • Console: The textual interface for message output and diagnostics.

At the top of the editor is a menu with several functional buttons:

  • Open / Save: Allows you to open and save Matita programs as text files with a .txt extension.
  • Download: Allows you to save the entire development environment as a single HTML file on your computer. This allows you to work locally, without the need for an internet connection. Furthermore, working locally makes it possible to use images saved on your device.
  • Change Language (EN/IT/NL): Changes the interface language. Automatically translates the entire source code written in the editor from English to Italian or Dutch and vice versa, preserving comments and text strings.
  • Help: Opens the technical documentation in a new window.
  • Run: The main button. Sends the order to the computer to interpret and execute the instructions.

Keyboard Shortcuts

To speed up writing and testing code, the IDE supports several useful keyboard shortcuts:

  • Ctrl + Enter: Equivalent to the Run button. Immediately starts the interpretation of the current program.
  • Ctrl + Space: Activates the auto-suggestions system (IntelliSense). If typed while writing a word, it opens a list of commands compatible with what you have typed.
  • TAB: If the suggestions menu is open, accepts and automatically inserts the selected command. It is also useful for indenting code and keeping it tidy.
  • ESC: Closes the suggestions window or other modal windows.

First Commands: write and clear screen

The write command sends text messages to the console. Consider the following example:

write "Hello world"

The console will output:

Hello world

Matita executes the program by interpreting the instructions sequentially, from first to last:

write "A"
write "B"

The previous code will produce the output AB.

To insert a line break, use the newline command:

write "A"
newline
write "B"

The special escape character \n performs the same function as newline within a string. The previous program is equivalent to:

write "A\n"
write "B"

You can concatenate multiple strings using the + operator. Example:

write "Hello " + "World.\n"

Formatting and Columns

The write command can be extended to format the output into columns, a very useful feature for creating tidy tables. By adding a colon : and a number, you reserve an exact number of spaces for the text. You can add a second modifier for alignment: c for center, l for left-align, r for right-align.

  • write:10: Reserves 10 spaces and right-aligns (default behavior).
  • write:10:c: Reserves 10 spaces and centers the text.
  • write:10:l: Reserves 10 spaces and left-aligns.
write:10:l "Name"; write:6:r "Grade"; newline
write:10:l "Alex"; write:6:r 8.5; newline
write:10:l "Maria"; write:6:r 10

To clear the console of all previous text, use the clear screen command.

clear screen
write "First line."
newline
write "Second line."

Comments V1.0.0

A fundamental syntactic element is the comment, which is ignored by the interpreter and is extremely useful for documenting or disabling parts of the code. Matita supports both the hash # and the double slash //:

# this is a traditional comment.
// this is also a valid comment.
put 31 in MarchDays // number of days in the month of March
write "March has " + MarchDays + " days."
Syntax Highlighting and Suggestions
Matita's IDE uses a color code to highlight the different syntactic elements of a program. It also offers suggestions as a mnemonic aid for using constructs.

Multiple commands on one line

To insert multiple instructions on the same line, you must separate them with a semicolon (;):

write "hello."; newline

Line Continuation V1.0.0

If an instruction is too long for a single line, it can be broken up using a backslash, i.e., the \ character at the end of the line. The interpreter will automatically join the lines ignoring the line breaks:

write \
"very long instruction written " + \
"on multiple lines for convenience"

Variables

A variable represents a memory area identified by a symbolic name (label), intended to hold a value (number, text, or complex data structures). Using variables allows you to write programs that operate on dynamic data, abstracting from the specific value contained.

To declare a variable and assign it a value (initialization), use the command: put ... in VariableName.

Consider the following example:

put 5 in A
put "Computer Science" in B
write "The value in A is " + A
newline
write B

The produced output will be:

 The value in A is 5
 Computer Science

To read the content of a variable, simply write the variable's name within an expression. In Matita, variable names are case-insensitive: Name, name, nAmE, NAME refer to the same memory location. A valid identifier must start with a letter and can contain numbers or the underscore _. Accented or special characters are not allowed.

To update the value of a variable, use the put command again.

put 100 in health
write "Initial health: " + health
newline
put health - 15 in health // The player takes 15 damage
write "State updated. Health is now: " + health

User Input: get

The get ... saying ... command activates a dialog box (prompt) that allows the user to enter data via the keyboard. The entered value is stored in the specified variable.

get PlayerName saying "Enter username:"
write "Registered name: " + PlayerName + ". Procedure started."

Mathematics

Arithmetic Operations and Precision

Expressions can be evaluated and assigned directly to variables.

put 10 in A
put 5 in B
put A+B in Sum
put A*B in Product
write "A + B = " + Sum; newline
write "A * B = " + Product

Calculating the remainder of a division (Modulo operation) is done using the word mod, or %, or the new safe variant %%.

clear screen
put 39 in A
put 7 in B
put A %% B in C
write "The remainder of " + A + "/" + B + " is " + C  // Result: 4

Operators and Expressions

In Matita, an expression is a combination of operands (numbers, texts, lists, points) and operators.

  • Boolean values: true, false
  • Logical operators: and, or, not
  • Comparisons:
    • Equality: =, ==, ===, equals, is
    • Inequality: <>, !=, !==, different from
    • Numeric comparisons: <, >, <=, >=
  • Arithmetic operators: +, -, *, /
  • Remainder (Modulo): mod, %, %%
  • Power: ^ or **
  • Negation (unary): -X

Power example:

put 2**3 in Cube
write Cube // Output: 8

Mathematical Constants and Functions V1.0.0

Matita 1.0.0 includes multiple constants and advanced mathematical functions, very handy for physics and geometry:

  • Constants: pi (or π), infinity, nan (Not a Number)
  • Geometric and calculation functions:
    • abs(x): Absolute value.
    • sin(x), cos(x), tan(x): Trigonometry (radians).
    • atan2(y, x): Arctangent (in radians, useful for calculating vector angles).
    • squareroot(x): Square root.
    • log(x), log10(x): Logarithms.
    • sign(x): Returns -1, 0, or 1 depending on the sign of the number.
  • Rounding functions:
    • round(x): Rounds to the nearest integer.
    • trunc(x): Removes the decimal part.
    • ceil(x): Rounds up to the next integer (upwards).

Rounding example:

write trunc(3.9) // Output: 3
newline
write ceil(3.1)   // Output: 4

Points and Coordinates: point

In addition to simple numbers, Matita allows you to work with Cartesian coordinates via the point object. This is essential for graphics but can also be used for vector or geometric calculations.

  • point(x, y): Creates a point object with the specified coordinates.
  • x of P / y of P: Accesses the horizontal and vertical coordinate of point P, respectively.
put point(10, 50) in P1
put point(20, 50) in P2

# Distance calculation (simplified, only on X axis)
put (x of P2) - (x of P1) in Distance
write "Horizontal distance: " + Distance 
# Output:  Horizontal distance: 10

Random Number Generation

Generating random numbers is essential for simulations and games. Matita offers several options via the random and dice commands:

  • random(): Without arguments, it returns a real number between 0.0 (inclusive) and 1.0 (exclusive).
  • random(MAX): Returns a real number between 0.0 and MAX.
  • random(MIN, MAX): Returns a real number between MIN and MAX.
  • dice(N): The command simulates rolling a die with N faces, so it returns an integer between 1 and N, inclusive.

Examples:

put random() in Probability // e.g. 0.453
put random(100) in Percentage // e.g. 87.21
put random(10, 20) in Temperature // e.g. 14.5
put dice(6) in Roll // e.g. 4 (gives an integer between 1 and 6)

Defining Numeric Precision

To control the number of decimal places displayed in the write command, you can use the digits command. This command only affects the display of the output in the console, not the internal value of the variable.

digits 2
put 10 / 3 in Result
write Result // Output: 3.33
newline
digits 5
write Result // Output: 3.33333

Scientific Notation: scientific

When Matita handles very large numbers, the system can automatically switch to displaying in scientific notation to improve readability. The format used is "number e exponent".

By default, the threshold for activating scientific notation is 1e6.

You can change this threshold using the scientific(N) command.

If you also set a value for digits, this will affect the number of digits shown in the mantissa.

// Set a low threshold to see the effect immediately
scientific 100 
put 150 in Value

write Value 
// Output:  1.5e2

// Change the displayed decimals in the mantissa
digits 4
write Value
// Output:  1.5000e2

Operator Precedence

  1. or
  2. and
  3. Comparisons (equals, different from, <, etc.)
  4. +, -
  5. *, /, mod, %%
  6. ^, ** (Power)
  7. Property or item access (e.g., x of P, item N of LIST)

Lists

The list is the fundamental data structure in Matita that allows you to store an ordered sequence of items. Unlike a simple variable, which holds a single piece of data, a list contains multiple pieces of data accessible via their position, called an index.

Lists in Matita are heterogeneous, meaning they can contain different types of data, and dynamic, meaning their size can change during execution.

Creation

To instantiate a list, you group the items within parentheses:

put (10, 20, 30) in myList

For greater semantic clarity, you can use the list keyword:

put list(10, 20, 30) in myList
put list() in EmptyList

List Operations

Matita provides a set of instructions for list manipulation.

Accessing Individual Items

To read a value at a specific position, use the item ... of ... syntax.

Indexing in Matita is 1-based: the first item is at index 1, the second at index 2, etc.

put (100, 200, 300) in data
put item 2 of data in Value // Value now contains 200
write Value // Prints 200

Searching for an item: To find the position of a specific value within the list, use the indexof(list, value) command. If the item is found, its index is returned (starting from 1), otherwise 0 is returned.

put ("Apple", "Pear", "Banana") in Fruit
put indexof(Fruit, "Pear") in Position
write "The Pear is at position " + Position // Output: 2

Determining the Dimension

To obtain the cardinality of the list (number of items), use the dimension of operator.

put ("red", "green", "blue", "yellow") in ColorList
put dimension of ColorList in N // N now contains 4
write ColorList + " are " + N + " colors\n"
write "the second color is " + item 2 of ColorList

The output will be:

 (red, green, blue, yellow) are 4 colors
 the second color is green

Maximum and minimum of a list of values

Matita offers functions that return the extreme values within a numeric list.

  • min(list): Returns the lowest value.
  • max(list): Returns the highest value.

When working with "hybrid" lists that contain both numbers and text, or when you need to know not only the extreme value but also its position (index), use the advanced functions:

  • valmin(list) / valmax(list): Calculate minimum and maximum ignoring non-numeric values (like strings or nulls).
  • argmin(list) / argmax(list): Return the index (position) where the minimum or maximum value is located, respectively.

Below is a complete example of how to use these functions to analyze a dataset.

// Let's introduce the data to work on.
// Note that the list of heights is hybrid, because it doesn't contain 
// only numeric values: the header, in fact, is a string.

put (("Name", "Height"),\ 
("Mark", 175), ("Anna", 182), ("Luke", 185), ("Julia", 176), ("Paul", 189),\
("Sofia", 168), ("David", 178), ("Helen", 165), ("Matthew", 181),\
("Claire", 173) ) in Group

// Let's create the list of heights
put () in HeightList // Initialization
for each P in Group you do ( add item 2 of P to HeightList )

// Let's find where the shortest and tallest members of the group are respectively:
put argmin(HeightList) in MinIndex
put argmax(HeightList) in MaxIndex

// Let's retrieve the names:
put item 1 of (item MinIndex of Group) in ShortName
put item 1 of (item MaxIndex of Group) in TallName

// Finally, output:
clear screen
write "Shortest member: " + ShortName + " (" + valmin(HeightList) + ")"; newline
write "Tallest member:  " + TallName  + " (" + valmax(HeightList) + ")"

Expected output:

 Shortest member: Helen (165)
 Tallest member: Paul (189)

Modification

Lists are mutable through specific commands.

Adding items: The add command inserts a new item at the end of the list. Syntax: add <value> to <listName>

put list(1, 2) in numbers
add 3 to numbers // numbers is now (1, 2, 3)

Inserting at a specific position: To insert a value at an arbitrary position, shifting subsequent items, use insert ... at item ... of .... Syntax: insert <value> at item <index> of <listName>

put ("A", "C") in Letters
insert "B" at item 2 of Letters
write Letters
// Output:  (A, B, C)

Removal at the head or tail:

put ("red", "green", "blue", "yellow") in ColorList
remove first item from ColorList
write ColorList
newline
remove last item from ColorList
write ColorList

Output:

 (green, blue, yellow)
 (green, blue)

Removal by index:

clear screen
put ("A","B","C","D") in LetterList
remove item 3 from LetterList
write LetterList

Output: (A, B, D)

Updating an item: To overwrite a value at a given position, combine put with item access.

put (10, 20, 30) in Data
write Data; newline
put 99 in item 2 of Data
write Data

Output:

 (10, 20, 30)
 (10, 99, 30)

Iteration

To execute a block of instructions for each item in the list, use the for each construct. The specified variable (e.g., n) sequentially takes the value of each item in the list. Note: The iterator variable contains a copy of the value. Modifying n within the loop does not modify the original list.

put (1, 2, 3) in Numbers
for each n in Numbers you do (
  put n * 2 in Double
  write Double
  newline
)
// Output:
//  2
//  4
//  6

Dictionaries, Records, and Sets

In addition to lists, Matita offers more complex data structures to manage information in a structured way: Dictionaries, Records, and Sets.

Dictionaries

Dictionaries (or "maps") allow you to store values by associating them not with a numeric index, but with a keyword (a string). They are defined using curly braces {}.

Creation and Initialization:

// Create an empty dictionary
put {} in Player

// Create an already populated dictionary
put {"name": "Alex", "lives": 3} in Player

Reading and Writing: You access a dictionary value using square brackets [].

write Player["name"] // Output: Alex
newline
// Let's update a value or add a new key
put 4 in Player["lives"]
put 1500 in Player["score"]

Operations on Dictionaries: Matita provides useful commands to manipulate this structure.

  • has key "K" in D: Returns true if the key is present.
  • keys of D: Returns a list containing all the keys of the dictionary.
  • dimension of D: Indicates the number of key-value pairs.
  • remove key "K" from D: Deletes the key and its value from the dictionary.
if (has key "score" in Player) then (
  write "The player has a score!"
  remove key "score" from Player
)

Records

Records offer a formal and compact way to create custom data types with named fields. Unlike dictionaries, in records fields (the "keys") are declared in advance, and access is done with the dot operator ..

To use a record, you must first declare its structure with the record command, and then you instantiate copies of it passing the parameters in order.

// Structure definition
record Person(name, age, city)

// Instantiation of a record (creation of a "copy")
put Person("Julia", 28, "Rome") in User1

// Reading and writing fields via the dot operator
write User1.name + " is " + User1.age + " years old."
put 29 in User1.age // Updates the value

Sets

Sets are unordered collections that, unlike lists, do not contain duplicates.

They are created with the set(...) command. They are particularly useful in combination with set mathematics to compare elements (e.g., finding intersections or unions using the standard operators `+`, `-`, `*` which have been overloaded for sets).

put set(1, 2, 2, 3) in S1
write S1 // Output: set(1, 2, 3) (duplicates are ignored)

Flow Control

if and otherwise if Construct

Conditional control structures allow you to execute blocks of code only when certain conditions are met. The fundamental instruction is if. The syntax is: if (CONDITION) then ( ... ) otherwise ( ... ).

For conditions, comparison operators are used: =, ==, equals, is, >, <, >=, <=, <>, !=, !==, different from.

You can chain multiple conditions using the otherwise if command:

clear screen
get Age saying "How old are you?"

if (Age >= 18) then (
  write "You are an adult."
) otherwise if (Age > 12) then (
  write "You are a teenager."
) otherwise (
  write "You are a child."
)

Loops

Loops allow you to iterate a block of instructions multiple times, optimizing code writing.

Simple Iteration: repeat

If you need to perform an action a specific number of times without the need for a variable index, you can use the repeat ... times construct.

repeat 3 times (
  write "Hooray!"; newline
)

Defined Iteration: index

To execute instructions needing a counter, use the index loop. A local variable (often named i) is created and automatically incremented from the start value up to the end value.

Syntax: index i goes from START to END and you do ( ... )

index i goes from 1 to 5 and you do (
  write "Iteration number " + i
  newline
)

Output:

 Iteration number 1
 Iteration number 2
 Iteration number 3
 Iteration number 4
 Iteration number 5

Undefined Iteration: as long as

This loop is executed as long as the specified condition remains true. It is suitable when the number of iterations is not known in advance.

Syntax: as long as (CONDITION) you do ( ... )

Countdown example:

clear screen
put 10 in countdown
as long as (countdown > 0) you do (
  write countdown
  newline
  put countdown - 1 in countdown
  wait 1000 ms
)
write "Start."

The program decrements the variable every second and terminates when the condition countdown > 0 becomes false.

Interrupting execution: stop

The stop command immediately halts the execution of the entire program. It is often used within a conditional structure to terminate a loop or a game when a certain condition is met.

if (LivesLeft equals 0) then (
  write "Game Over"
  stop
)

Time Management

Waiting time: wait

The execution of instructions is sequential and immediate. However, in contexts such as animations, it is necessary to introduce pauses. The wait command suspends execution for a given interval. The unit of measurement is the millisecond (ms). Example:

wait 45 ms

This instruction forces the processor to pause for 45 milliseconds before moving to the next instruction.

Stopwatch: tick and tock

To measure elapsed time, Matita provides an integrated stopwatch.

  • tick: A command that starts or resets the stopwatch at the current moment.
  • tock: A special variable that contains the number of milliseconds elapsed since the last tick command.

Measurement example:

write "Measurement started..."
newline
tick // Starts the stopwatch
wait 1500 ms
write "Elapsed time: " + tock + " ms"

Note: If you try to read the value of tock without first having executed tick, the program will report an error.

Data Manipulation

Matita offers simple commands to analyze and transform strings and numbers by treating them as lists of characters or digits.

Transforming a string into a list

The decomposition command divides a string or a number into its individual characters, returning a list.

put "Hello" in Text
put decomposition(Text) in CharacterList
write CharacterList
// Output:  (H, e, l, l, o)

Splitting by separator: split

Unlike decomposition, which separates every single character, the split(text, separator) command divides a string based on a specific character (such as a space or comma).

put "red,green,blue" in CSV
put split(CSV, ",") in Colors
write Colors
// Output:  (red, green, blue)

Specific Decomposition

The decomposition(value, TYPE) command offers more control. The available target types are: DIGITS, CHARACTERS, and WORDS.

// Split a sentence into words
put "Matita is fun" in Sentence
put decomposition(Sentence, "WORDS") in WordList
write WordList
// Output:  (Matita, is, fun)

// Split a number into digits
put 123.45 in Number
put decomposition(Number, "DIGITS") in DigitList
write DigitList
// Output:  (1, 2, 3, ., 4, 5)

Composition

The inverse operation to decomposition is composition(list, TYPE), which joins the elements of a list into a single value. The available target types are TEXT and NUMBER.

put ("S", "e", "a") in Letters
put composition(Letters, "TEXT") in Word
write Word
// Output:  Sea

// Create a number from digits
put (3, ".", 1, 4) in Digits
put composition(Digits, "NUMBER") in Value
write Value * 2 // We can do calculations because it's a number
// Output:  6.28

Type Inspection

In Matita, variables can change type dynamically. You can check the nature of a datum using the following functions:

  • type(x): Returns a string describing the data type (e.g., "number", "string", "list", "point", "boolean").
  • numeric(x): Returns true if the argument is a valid and finite number, false otherwise.
put 42 in X
write type(X)        // Output: number
write numeric(X)     // Output: true

put "Hello" in Y
write numeric(Y)     // Output: false

Type Conversion (Casting)

Sometimes it is necessary to treat a number as if it were text, for example to concatenate it with other words, or conversely, to use it in a mathematical calculation. The cast command performs the type conversion operation.

  • cast(NUMBER, "string"): Returns a string containing the number.
  • cast(STRING, "number"): Returns a number (supports positive, negative, decimal integers and scientific notation).

Example:

clear screen
// Test from String to Number
put "10" in S1
put "3.14" in S2
put "-50" in S3
put "1.2e3" in S4  // 1200 in scientific notation

put cast(S1, "number") + cast(S2, "number") + cast(S3, "number") + cast(S4, "number") in Total
write "Sum of casts: " + Total 
newline

// Test from Number to String (Concatenation)
put 500 in N1
put 12.5 in N2
// Add the strings (concatenation) instead of numbers
write "Concatenation: " + cast(N1, "string") + cast(N2, "string")

// Expected Output:
//  Sum of casts: 1163.14
//  Concatenation: 50012.5

Unicode Function

Unicode is an international standard that assigns a unique numeric code to every character or symbol used in computer systems, including letters of all alphabets, and Emojis.

The unicode(x) command acts as a bridge between numbers and symbols:

  • From Number to Symbol: If the argument is a number, it returns the associated character.
  • From Symbol to Number: If the argument is a string, it returns the numeric code of the first character.

Example:

// Example 1: Generate a symbol from the code (Rocket)
put unicode(128640) in RocketSymbol
write "To infinity... " + RocketSymbol
// Output:  To infinity... 🚀
newline

// Example 2: Get the code from a symbol (Smiley)
put unicode("🙂") in SmileyCode
write "The smiley code is: " + SmileyCode
// Output:  The smiley code is: 128578

Procedures and Functions

Using loops allows you to solve many problems. However, as complexity increases, it becomes necessary to organize the code in a modular way to promote readability and maintainability.

Matita supports procedural programming by allowing the definition of custom commands. To avoid code duplication and encapsulate logic, we use Procedures and Functions:

  • A Procedure is a block of instructions that performs a well-defined task. Instructions that can be used in multiple places in the code are grouped into procedures. Procedures can apply the same operations on different parameters, provided as input. A procedure executes instructions but does not produce an output value.
  • A Function performs operations and returns a value (output) that can be used in other expressions via the return command.

Important: The Matita interpreter reads and executes the code line by line. Therefore, it is essential that each procedure or function is defined before the line where the first call occurs.

Procedures

Example of defining and calling a procedure:

// Definition
define procedure greet(name) (
  write "Hello, " + name + ". Welcome to Matita."
  newline
)

// Call
greet("Alex")
greet("Maria")

Functions

Example of defining and calling a function:

define function Double(number) (
  put number * 2 in result
  return result
)

put 7 in Number
put Double(Number) in Doubled
write "The double of " + Number + " is " + Doubled
newline
Identifying constructs vs built-in functions. Some expressions are identifying: they refer to a part or property of something that already exists (e.g., x of pointP, item 3 of myList, dimension of myList). Other expressions are generative and produce a new value (built-in functions), for example point(10,20), list(1,2,3), sin(angle), squareroot(9).

Recursion

Recursion is a programming technique in which a function calls itself. It is particularly effective for solving problems that can be broken down into identical subproblems of a smaller size. The classic example is the calculation of the factorial (𝑛!).

define function factorial(n) (
  // Base case: termination condition.
  if (n equals 0) then (
    return 1
  ) otherwise (
    // Recursive step
    return n * factorial(n - 1)
  )
)

put 5 in Number
put factorial(Number) in Result
write Number + "! = " + Result // Will show 120

If the chain of recursive calls is excessive, a stack overflow occurs (recursion limit exceeded).

Memory Management

Variable Scope

Scope is the visibility area of a variable. In nested code blocks (inside you do, as long as, if, for each, etc.), the variables created with put are local to the block itself.

Variables defined in the main flow are called global. A global variable is readable within a local block, but if you try to assign a value to a variable with the same name in the local block, a new local variable is created that "hides" the global one (shadowing). In these cases, Matita issues a warning:

 Warning: Local variable 'Number' is shadowing a global variable with the same name.

Consider the following example:

clear screen
put 1 in Number
as long as (Number < 6) you do (
  put Number + 1 in Number
  write Number
  newline
)
// we are now outside the local block
write "Final value: " + Number

Output:

 Warning: Local variable 'Number' is shadowing a global variable with the same name.
 2
 3
 4
 5
 6
 Final value: 1

As can be seen, the Number variable inside the loop is distinct from the external one. At the end of the loop, the global variable retains its original value, which is 1.

The global directive

To modify a global variable inside a local scope, you must explicitly declare it with the global directive:

clear screen
put 1 in Number
global Number
as long as (Number < 6) you do (
  put Number + 1 in Number
  write Number
  newline
)
// we are now outside the block
write "Final value: " + Number

Output:

 2
 3
 4
 5
 6
 Final value: 6

You can declare multiple global variables simultaneously by separating them with a comma:

global Name1, Name2 // multiple variables separated by commas

Value Semantics

The Matita interpreter adopts, by default, a Value Semantics. Every assignment operation via the put command involves a Deep Copy of the data.

This means that assigning a list or a complex object to a new variable creates an independent and distinct clone in memory. This prevents unexpected alterations and makes programming safer and more understandable, especially for educational purposes.

put (1, 2, 3) in A
put A in B      // B is a local copy of A
put 99 in item 1 of B
// A remains (1, 2, 3), B becomes (99, 2, 3)

Access by reference: alias

In various languages, including Python and Java, the instruction B = A causes both variables to point to the same object in memory. It is said that such languages implicitly adopt a reference semantics and A and B are aliases, that is, replicas of each other. An alias therefore does not contain its own data, but acts as a pointer to another target variable.

To create a shared reference in Matita, for example to allow data sharing or resource optimization, we introduce the concept of explicit aliasing.

An alias is created via an explicit request through the command: B alias A, where A is an existing variable.

Any read or write operation performed on B is intercepted by the interpreter and redirected to the target variable A. The following code:

put 4 in A
put 56 in (B alias A)
write "A: " + A + "; B: " + B

produces the output:

A: 56; B: 56

Pass by Reference to Procedures: inout

Matita supports two ways of passing parameters to a procedure.

  1. Pass by value (default): The value of the argument is copied into the local scope of the procedure. Modifications do not propagate to the caller.
  2. Pass by reference (using the inout keyword): Using the inout keyword, the interpreter creates a local alias that points to the variable passed by the caller.
define procedure Increment(inout X) (
    put X + 1 in X  // Directly modifies the external variable
)

Freeing memory: null

The keyword null indicates a special value corresponding to the absence of data. This keyword allows you to empty a variable.

For example, by writing put null in VAR, the name VAR is disconnected from its previous content, which will signal the computer that that content is no longer needed and the memory it occupied can be freed.

Using null is useful for freeing the computer's memory when heavy objects, such as images or very long lists, are no longer needed.

Object-Oriented Programming (OOP)

Object-Oriented Programming (OOP) is a paradigm that allows modeling software by grouping data and behaviors (variables and procedures) into unique entities, called objects. This approach makes the code more organized, modular, and closer to how we think about the real world.

Classes and Instances

A class is the "blueprint" or "mold" that defines the structure of an object. The concrete object created from that mold is called an instance.

To define a class, use the keyword class followed by the name and its fields (the internal variables that make up the state of the object). You can then create copies of the object by calling the class name as if it were a function:

// Class definition
define class Player (name, score) (
  // Methods (internal procedures and functions) will go here
)

// Creating an instance
put Player("Alex", 0) in P1
write P1.name // Output: Alex

Custom constructor: INITIALIZE

When an instance is created, Matita automatically assigns the passed values to the class fields in order. However, if you want to execute specific code at the time of creation (for example, to do calculations or validations), you can define a special procedure called INITIALIZE within the class.

Methods and Encapsulation (Privacy)

Procedures and functions defined inside a class are called methods. Methods can access class fields directly without having to pass them as parameters. To refer to the instance itself within methods, you can use the keyword this.

A good practice in OOP is encapsulation: hiding sensitive data from the outside and allowing its modification only through controlled methods. In Matita, any field that starts with an underscore (_) is considered private. If you try to access it directly from the outside, the program will not error out to avoid interrupting execution, but will print a Warning in the console to discourage its use.

define class BankAccount (owner, _balance) (
  
  define procedure INITIALIZE(t, s) (
    put t in owner
    put s in _balance
  )

  define procedure deposit(amount) (
    put _balance + amount in _balance  // OK: legal internal access
  )

  define function read_balance() (
    return _balance             // OK: legal internal access
  )
)

put BankAccount("Alice", 100) in MyAccount

MyAccount.deposit(50)
write MyAccount.read_balance() // Correct way: prints 150
newline

// Generates a "Warning on line: '_balance' is a private member..."
write MyAccount._balance

Inheritance (extends) and the SUPER directive

Inheritance allows you to create a new class based on an existing class (parent), inheriting its fields and methods. It is achieved using the keyword extends.

If the child class defines a method with the same name as the parent's, it overwrites it (override). To still invoke the original method of the parent class, use the keyword SUPER.

define class Animal (name) (
  define procedure sound() (
    write "I make a generic sound."
  )
)

// The Dog extends the Animal, inheriting the 'name' field
define class Dog extends Animal (breed) (
  define procedure sound() (
    write "Woof! My name is " + name + ". "
    SUPER.sound() // Calls the sound() method of Animal
  )
)

put Dog("Fido", "Beagle") in MyDog
MyDog.sound()
// Output: Woof! My name is Fido. I make a generic sound.

Polymorphism: classof()

To find out which class an instance belongs to during program execution (useful for example in collisions between objects in a video game), use the classof(object) function. It returns the name of the class as a lowercase text string.

if (classof(MyDog) equals "dog") then (
  write "It's definitely a dog!"
)

Part Two: Graphics and Interactivity

Graphical Elements

The graphics area uses a Cartesian coordinate system. The position of each element is defined by a pair of coordinates (x, y). The origin (0, 0) is located at the center of the area. Positive values of x indicate positions to the right, negative values to the left. Positive values of y indicate positions upwards, negative values downwards.

Coordinates are represented as lists of two numbers. You can use the explicit syntax point(x, y). To access the individual components of a point, you use the x of and y of properties.

put point(10, 44) in MyPoint
put x of MyPoint in CoordinateX
write CoordinateX

To change a single coordinate, use the set command. This operation is fundamental for managing object movement.

// Move the point 10 steps down without changing the x
set y of MyPoint to (y of MyPoint - 10)
write y of MyPoint // Will show 34

Colors and Backgrounds

Matita uses the RGB color model. Colors are defined via a list of three values (R, G, B), respectively representing the percentages of Red, Green, and Blue, ranging from 0 to 100, inclusive.

Example of setting the background:

put (68, 85, 90) in skyBlue
set color skyBlue
paint background

To erase the content of the graphics area, for example before drawing a new frame in an animation, use the clear graphics command.

Drawing Basic Shapes

After setting the current color, you can draw graphical primitives.

Circle: Syntax circle radius R at POS. Draws a filled disk.

set color (100, 87, 0)
circle radius 60 at (120, 80)

Circumference: Syntax circumference radius R thickness S at POS. Draws only the outline.

set color (0, 0, 0)
circumference radius 25 thickness 4 at (-200, -50)

Line: Syntax line thickness S from P1 to P2. Draws a segment between two points.

set color (0, 0, 0)
line thickness 2 from (-200, -75) to (-200, -160)

Triangle: Syntax triangle with vertices P1, P2, P3. Draws a filled triangle connecting the three specified points separated by commas.

set color (0, 50, 100)
triangle with vertices (0, 50), (-50, -50), (50, -50)

Arc: Syntax arc radius R thickness S from P1 to P2. Draws a circular arc of a given radius, passing through the two given points. Since there are at most two circles of a given radius passing through two assigned points in the plane, the code will draw the arc that goes from the first to the second point rotating counterclockwise. This implies that if you invert the order of the two points, you will get an arc with an inverted concavity.

set color (100, 0, 0)
arc radius 240 thickness 12 from (-220, -160) to (220, -160)

Composite example:

clear graphics

put 15 in S       // Thickness of each arc
put 260 in R      // Radius of the outermost color (Red)
put -160 in Y     // Horizon height (where the grass ends)

// --- SKY AND BACKGROUND ---
set color (68, 85, 90)
paint background

// --- GRASS ---
set color (16, 65, 27)
rectangle from (-360, -250) to (360, Y)

// --- SUN ---
set color (100, 87, 0)
circle radius 70 at (250, 180)

// --- RAINBOW ---
// 1. Red
set color (90, 0, 0)
arc radius R thickness S from (R, Y) to (-R, Y)
put R - S in R  // Shrink the radius for the next color

// 2. Orange
set color (90, 50, 0)
arc radius R thickness S from (R, Y) to (-R, Y)
put R - S in R

// 3. Yellow
set color (90, 90, 0)
arc radius R thickness S from (R, Y) to (-R, Y)
put R - S in R

// 4. Green
set color (0, 70, 0)
arc radius R thickness S from (R, Y) to (-R, Y)
put R - S in R

// 5. Blue
set color (0, 0, 90)
arc radius R thickness S from (R, Y) to (-R, Y)
put R - S in R

// 6. Indigo / Violet
set color (40, 0, 70)
arc radius R thickness S from (R, Y) to (-R, Y)

// --- BALLOON ---
set color (80, 0, 90) // A brighter purple
circle radius 25 at (-100, -50)
line thickness 2 from (-100, -75) to (-100, Y)

Graffiti (Graphical Text)

The graffiti command allows you to insert text directly into the graphics area.

set color (10, 10, 10)
graffiti size 30 at point(0, 0) "Center."

Redrawing a scene

The redraw command forces the update of the graphics area (canvas). This command is essential to display the changes that occurred in the scene stored in memory.

Framing graphics (Auto-framing)

The frame graphics command automatically adjusts the camera (pan and zoom) so that all objects currently present in the graphics area (canvas) are visible and centered in the window.

This command is useful for making the scene dynamically fit the screen. It also implicitly performs a redraw, so its use makes it unnecessary to use redraw.

set color (100, 0, 0)
// Draw a very large circle, far from the origin
circle radius 500 at (2000, 2000)
// Adjust the view to show it all
frame graphics

Creating Animations

To achieve movement and animation of objects in Matita, we exploit the principle of persistence of vision: a series of static images, slightly different from each other, shown in rapid succession, creates the illusion of movement.

The typical structure of an animation loop involves the following steps:

  1. Start a loop (usually as long as (true)).
  2. Clear the previous graphics (with clear graphics or paint background).
  3. Update the coordinates of the objects (for example, moving a point).
  4. Draw the objects in the new positions.
  5. Force the graphical update with redraw (optional if you use wait, which does it implicitly).
  6. Wait for a short time interval (e.g., wait 16 ms for about 60 frames per second).

Example of a simple animation:

put -150 in X
global X
as long as (X < 150) you do (
  put X + 2 in X
  clear graphics
  circle radius 20 at point(X, 0)
  wait 16 ms
  redraw
)

Keyboard and Mouse Input

Keyboard

To create real-time interactions, you check the state of the keys. The expression key "KEY_NAME" is pressed returns true if the indicated key is currently pressed. The directional keys are mapped as UP, DOWN, LEFT, RIGHT.

Other control keys: SPACE, ENTER, ESC, TAB, BACKSPACE, SHIFT, CTRL, ALT. Alphanumeric characters are simply indicated by their symbol ("A", "x", etc.).

Mouse

Matita also allows you to detect the position of the mouse pointer and the state of the main button:

  • mouse_x() and mouse_y(): Return the coordinates in the graphical plane based on the cursor position.
  • mouse_pressed(): Returns true as long as the left mouse button is held down on the canvas.

Example of controlling an object via keyboard:

put point(0,0) in playerPos
put 300 in lim

as long as (true) you do (

  if (key "UP" is pressed) then (
    set y of playerPos to (y of playerPos + 5)
  )
  if (key "DOWN" is pressed) then (
    set y of playerPos to (y of playerPos - 5)
  )
  if (key "RIGHT" is pressed) then (
    set x of playerPos to (x of playerPos + 5)
  )
  if (key "LEFT" is pressed) then (
    set x of playerPos to (x of playerPos - 5)
  )

  // Emergency exit
  if (key "x" is pressed) then (stop)

  // Drawing
  clear graphics
  set color (0, 100, 100)
  rectangle from (-lim, -lim) to (lim, lim)

  set color (0, 50, 100)
  circle radius 20 at playerPos

  redraw

  // Refresh rate (about 60 fps)
  wait 16 ms
)

Graphical Labels

To optimize code and reuse groups of shapes, you can define labels. A label groups a series of drawing instructions into a single reusable object. Syntax: label (...) as NAME.

Example of definition and use:

// Model definition
label (
  // Body
  set color (100, 100, 100) // White
  circle radius 30 at (0, 0)
  circle radius 20 at (0, 45)
  circle radius 15 at (0, 75)

  // Details
  set color (0, 0, 0) // Black
  circle radius 2 at (-5, 78)
  circle radius 2 at (5, 78)
  circle radius 2 at (0, 50)
  circle radius 2 at (0, 40)

  // Nose
  set color (100, 60, 0) // Orange
  triangle with vertices (0, 75), (0, 73), (8, 74)
) as Snowman

set color (50,50,50)
paint background
draw Snowman at point(150, 0)
draw Snowman at point(-100, 0)

Transformations: Scale and Rotation

Labeled objects can undergo geometric transformations.

Scaling

The scale command resizes the label. A factor of 2 doubles the dimensions, 0.5 halves them.

Note: The scaling operation is permanent for the label to which it is applied.

scale Snowman by 0.5
draw Snowman at point(0, 100)
draw Snowman at point(0, -100)

Rotation

It is possible to rotate an object with respect to the center of its bounding box. The angle is expressed in sexagesimal degrees (0-360), with counterclockwise rotation for positive values.

rotate Snowman of 45 deg and place it at (300, -300)

Images

It is possible to import external image files to use them as graphical objects or backgrounds. The files must reside in the same directory as the matita.html file.

Images as Labels

A loaded image is treated as a label and can therefore be drawn, scaled, and rotated.

import "FileName.png" as kitten
draw kitten at (0, 0)
scale kitten by 0.5
rotate kitten of 15 deg and place it at (-120, -60)

Images as Backgrounds

To set an image as the background of the entire graphics area:

load "grass.png" scale 0.5 as background
paint background

Importing External Code (Libraries)

It is possible to divide complex programs into multiple files. Using the import command without the keyword as, Matita will read a text file containing source code (e.g., useful procedures or functions) and execute it as if it had been written directly in the main file.

import "my_library.txt"
// Now you can use functions defined in my_library.txt

Audio and Sounds

Matita integrates a sound synthesis engine to add audio effects and music (ADSR synthesizer). It is based on commands to define the physical properties of the sound, play it, control its volume, and apply fades.

Defining and playing a sound

You use the sound command followed by a name of your choice, then open a configuration block. The main configurations can generate a noise, a continuous frequency sweep, or a sequence of notes.

// Creates a simple beep (sweep from 800Hz to 200Hz)
sound Beep (
  sweep from 800 to 200 hz
  wave "sine" // Options: "sine", "square", "sawtooth", "triangle"
  duration 300 ms
)

// Creates a noise (great for explosions)
sound Explosion (
  noise
  duration 500 ms
  decay 400 ms
)

// Creates a melody
sound Melody (
  notes (440:200, 523:400, 659:600) // Frequency : Duration
  wave "square"
)

// Playback
play Beep
wait 500 ms
play Explosion

Looping and Controls

By adding the repeat true parameter to a sound's configuration, it will continue to play indefinitely. You will need to use control commands to stop it.

  • play NAME: Plays the sound.
  • stop NAME: Immediately stops the playback of the sound.
  • volume NAME 50: Adjusts the volume of the sound (from 0 to 100).
  • fade NAME out 1000 ms: Smoothly reduces the volume to 0 in the specified time. Use `in` instead of `out` to fade the volume in.
  • is playing NAME: Function that returns `true` if the sound is currently playing.
sound Engine (
  noise
  repeat true
)

play Engine
volume Engine 20
wait 3000 ms
fade Engine out 1500 ms

Interacting with the canvas

In addition to rendering the graphics generated by the code, Matita's graphics area (the canvas) offers interactive tools to inspect the drawing space and locate points.

Pan and Zoom

You can navigate inside the graphics area without altering the original coordinates of the objects:

  • Pan (Dragging): By holding down the right mouse button (or the middle button, or Shift + left button) and moving the cursor, you can translate the view in any direction.
  • Zoom: By using the mouse wheel, you zoom in or out of the view. The zoom is centered on the cursor, i.e., it points towards the coordinates where the mouse is at that moment.

Orthogonal Constraints (Ortho) and Magnet (Snap)

The buttons located on the top bar of the canvas window allow you to activate geometric precision tools:

  • Magnet (Snap): If active, the cursor is attracted and snapped to notable points of already drawn objects (centers of circles, vertices of rectangles, midpoints of lines, etc.).
  • Ortho (⟂) and Alpha Angle (α): By activating the Ortho tool, it is possible to draw lines to support alignment. With a single click on the canvas, you fix a base point; moving the mouse, the crosshair will be constrained to move only horizontally or vertically. By clicking the α button and entering a numerical value in the field next to it, you can freely rotate the orthogonal axes by that number of degrees. A second click releases the base point.
  • Save PNG: Downloads the contents of the canvas as a .png image.
  • Fit to Window (⛶): Adjusts the zoom and position of the camera to frame all the objects within the graphics window.

Transfer of Coordinates

To speed up writing code and positioning objects, Matita allows you to grab positional data directly from the graphics area and insert it into the text editor.

By performing a long click with the left mouse button on a point on the canvas, the coordinates of that point (for example, (15.2, 30.5)) will be automatically typed into the Code Editor, at the text cursor. If this operation is performed while the Magnet is running and is snapped to a point whose coordinates are saved in a global variable, the name of that variable will be pasted directly into the Editor instead of the numerical values, preserving the consistency of the code.
